# 구글 시트 연동 + 보안 설정 가이드

## ⚠️ 기존 권한 점검 결과 (중요)

처음 배포했던 버전은 웹앱을 "전체 접근 허용"으로 배포했기 때문에, **exec URL만 알면 누구나
읽기/쓰기가 가능한 상태**였습니다 (사이트를 거치지 않고 URL에 직접 요청해도 출석·말씀나눔
데이터를 조작할 수 있었음). 이번 업데이트로 아래 두 가지를 추가해 이 문제를 막았습니다.

1. **서버 측 키 검증** — Code.gs 가 SITE_KEY(열람)/ADMIN_KEY(입력)를 확인하며, 키가 없거나
   틀리면 요청 자체를 거부합니다.
2. **화면 비밀번호 게이트** — 사이트 전체는 비밀번호를 입력해야 열람 가능, 체크 입력
   화면(admin.html)은 별도의 관리자 비밀번호가 있어야 접근 가능합니다.

단, 정적 사이트 특성상 이는 "완벽한 보안"이 아니라 **캐주얼한 접근을 막는 수준**입니다.
브라우저 개발자 도구를 다룰 줄 아는 사람이 작정하고 파고들면 키 값을 알아낼 수 있습니다.
민감도가 매우 높은 정보는 이 사이트에 올리지 않는 것을 권장합니다.

## 1. 구글 시트 만들기
1. [sheets.new](https://sheets.new) 로 새 스프레드시트를 만듭니다.
2. 이름을 "편집부 출석·말씀나눔 기록" 정도로 정합니다.

## 2. Apps Script 붙여넣기
1. 시트 메뉴에서 **확장 프로그램 → Apps Script** 를 클릭합니다.
2. 열린 편집기의 기본 코드(`Code.gs`)를 모두 지우고, 이 폴더의 `Code.gs` 파일 내용을
   전체 복사해서 붙여넣습니다.
3. 맨 위쪽의 두 줄을 원하는 값으로 바꿉니다 (기본값은 꼭 변경하세요!):
   ```js
   var SITE_KEY = "hanmaeum2026";       // 사이트 비밀번호
   var ADMIN_KEY = "editor-admin-77";   // 관리자 비밀번호
   ```
4. 저장 아이콘(💾)을 클릭합니다.

## 3. 웹 앱으로 배포하기
1. 우측 상단 **배포 → 새 배포**를 클릭합니다.
2. 유형 선택에서 톱니바퀴를 눌러 **웹 앱**을 선택합니다.
3. 설정값:
   - 설명: `editorial-site` (아무 이름이나 가능)
   - 다음 사용자 권한으로 실행: **나**
   - 액세스 권한이 있는 사용자: **전체 (익명 사용자 포함)**  ← 이건 그대로 두세요.
     (실제 접근 제어는 위에서 설정한 SITE_KEY/ADMIN_KEY가 담당합니다)
4. **배포**를 클릭하면 구글 계정 권한 승인 화면이 나옵니다. 본인 계정으로 승인해 주세요.
5. 배포가 끝나면 **웹 앱 URL**을 복사하세요. (`https://script.google.com/macros/s/...../exec`)

## 4. 사이트에 URL과 비밀번호 연결하기
`assets/js/config.js` 파일을 열고 아래 값을 채웁니다.

```js
GAS_URL: "여기에 웹앱 URL 붙여넣기",
SITE_PASSWORD_HASH: "...",   // 아래 5번에서 만든 해시값
ADMIN_PASSWORD_HASH: "...",  // 아래 5번에서 만든 해시값
```

기본값(비밀번호 "hanmaeum2026" / "editor-admin-77")을 그대로 쓰려면 이 파일은 건드리지
않아도 됩니다. 단, **꼭 비밀번호를 바꾸는 것을 추천**합니다 (아래 5번 참고).

## 5. 비밀번호 바꾸는 방법 (권장)
1. `gas/hash-tool.html` 파일을 더블클릭해서 브라우저로 엽니다 (인터넷 연결 없이도 동작).
2. 새 비밀번호를 입력하고 "해시 생성" 클릭 → 나오는 긴 문자열을 복사.
3. `assets/js/config.js` 의 `SITE_PASSWORD_HASH` 또는 `ADMIN_PASSWORD_HASH` 값에 붙여넣기.
4. `gas/Code.gs` 의 `SITE_KEY` 또는 `ADMIN_KEY` 값을 **원문 비밀번호**(해시 아님)로 변경.
5. Apps Script 편집기에서 **배포 → 배포 관리 → 수정(연필) → 새 버전 → 배포** 로 다시 배포합니다.
   (URL은 그대로 유지되니 config.js의 GAS_URL은 안 바꿔도 됩니다)

## 6. 확인하기
- 사이트 아무 페이지나 열면 비밀번호 입력창이 뜹니다 → 사이트 비밀번호 입력.
- `/editorial/admin.html` 접속 → 관리자 비밀번호 입력 → 출석/말씀나눔 체크 후 저장.
- 방금 만든 구글 시트를 열어보면 `attendance`, `scripture` 탭이 자동으로 생기고,
  체크한 내용이 한 줄씩 기록되어 있으면 성공입니다.
- 사이트의 "출석 현황" / "말씀 나눔 현황" 페이지에서 최근 30일 요약이 보이는지 확인하세요.

## 나중에 수정하고 싶다면?
- **담당(1면/2면) 및 일정**은 `assets/js/config.js` 의 `schedule` 배열을 수정하면 됩니다.
- **편집부원 명단**은 같은 파일의 `members` 배열을 수정하면 됩니다.
- 매 학기/분기마다 새로운 일정표가 나오면 이 배열에 이어서 추가해 주세요 (날짜순 유지).
- **admin.html 주소는 사이트 메뉴에 노출하지 않았습니다.** 담당 선생님만 북마크해서
  사용하세요. (`winsoul.de/editorial/admin.html`)

## 문제가 생기면
- 저장이 안 될 경우, 브라우저 개발자 도구(F12) → Console 탭에서 에러 메시지를 확인하세요.
- "인증 실패" 메시지가 뜨면 config.js의 해시와 Code.gs의 키가 서로 다른 비밀번호로
  설정되어 있는 경우가 대부분입니다. 5번 순서대로 다시 맞춰보세요.
- 코드를 수정한 뒤에는 항상 **새 버전으로 재배포**해야 반영됩니다.
