# 구글 시트 연동 설치 가이드 (5분이면 끝나요)

이 사이트의 "출석 체크"와 "말씀 나눔 체크"는 여러 선생님/기기에서 같은 기록을 보기 위해
구글 시트에 데이터를 저장합니다. 아래 순서대로 딱 한 번만 설정하면 됩니다.

## 1. 구글 시트 만들기
1. [sheets.new](https://sheets.new) 로 새 스프레드시트를 만듭니다.
2. 이름을 "편집부 출석·말씀나눔 기록" 정도로 정합니다.

## 2. Apps Script 붙여넣기
1. 시트 메뉴에서 **확장 프로그램 → Apps Script** 를 클릭합니다.
2. 열린 편집기의 기본 코드(`Code.gs`)를 모두 지우고, 이 폴더의 `Code.gs` 파일 내용을
   전체 복사해서 붙여넣습니다.
3. 저장 아이콘(💾)을 클릭합니다.

## 3. 웹 앱으로 배포하기
1. 우측 상단 **배포 → 새 배포**를 클릭합니다.
2. 유형 선택에서 톱니바퀴를 눌러 **웹 앱**을 선택합니다.
3. 설정값:
   - 설명: `editorial-site` (아무 이름이나 가능)
   - 다음 사용자 권한으로 실행: **나**
   - 액세스 권한이 있는 사용자: **전체 (익명 사용자 포함)**  ← 중요! 이걸 선택해야 사이트에서 접근 가능합니다.
4. **배포**를 클릭하면 구글 계정 권한 승인 화면이 나옵니다. 본인 계정으로 승인해 주세요.
   ("안전하지 않음" 경고가 떠도, 내가 만든 스크립트이므로 **고급 → 이동(안전하지 않음)** 을 눌러 진행하면 됩니다.)
5. 배포가 끝나면 **웹 앱 URL**이 나옵니다. (`https://script.google.com/macros/s/...../exec` 형태)
   이 URL을 복사하세요.

## 4. 사이트에 URL 연결하기
1. 사이트 파일 중 `assets/js/config.js` 파일을 엽니다.
2. 맨 위쪽의 아래 줄을 찾아서
   ```js
   GAS_URL: "",
   ```
   따옴표 안에 복사한 웹 앱 URL을 붙여넣습니다.
   ```js
   GAS_URL: "https://script.google.com/macros/s/AKfycb.../exec",
   ```
3. 파일을 저장하고, 사이트를 다시 업로드(또는 새로고침)하면 완료입니다.

## 5. 확인하기
- 사이트의 "모임 출석 체크" 또는 "말씀 나눔 체크" 페이지에서 아무 이름이나 체크하고 저장해 보세요.
- 방금 만든 구글 시트를 열어보면 `attendance`, `scripture` 탭이 자동으로 생기고,
  체크한 내용이 한 줄씩 기록되어 있으면 성공입니다.

## 나중에 코드를 수정하고 싶다면?
- **담당(1면/2면) 및 일정**은 `assets/js/config.js` 의 `schedule` 배열을 수정하면 됩니다.
- **편집부원 명단**은 같은 파일의 `members` 배열을 수정하면 됩니다.
- 매 학기/분기마다 새로운 일정표가 나오면 이 배열에 이어서 추가해 주세요 (날짜순 유지).

## 문제가 생기면
- 사이트에서 저장이 안 될 경우, 브라우저 개발자 도구(F12) → Console 탭에서 에러 메시지를 확인하세요.
- 웹 앱 배포 시 "액세스 권한: 전체"를 선택하지 않으면 외부에서 저장 요청이 거부됩니다.
- 코드를 수정한 뒤에는 **배포 → 배포 관리 → 수정(연필 아이콘) → 새 버전**으로 다시 배포해야
  변경사항이 반영됩니다 (URL은 그대로 유지됩니다).
